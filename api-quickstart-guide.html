API Quick Start Guide

This quick start guide demonstrates how to call the System Requirements Lab API to query for data from a user test on the System Requirements Lab Test App.

System Requirements Lab API

The API provides calls that query for user test data. Queries may be performed on date ranges or session and user IDs.

Test App

The Test App let users run tests of their computer and network against a set of requirements. During the test, information is gathered from the user's computer and network. At the test end, an analysis of the user's system against a set of requirements is produced.

Data

A user takes a test on the Test App and data is gathered. This data has three forms: Session, System, and Analysis.

Session
Session data comprises test date and time, passed in fields e.g., user ID, user entered information e.g., name and email address, and geo-location data.
System
System data comprises information of a user's computer and network e.g., CPU, RAM size, and internet speed.
Analysis
Analysis data comprises pass/fail test results of the user's system and requirements.

Access Token

Access tokens are used in token-based authentication to allow an application to access the SystemRequirementsLab API. Once an application has received an access token, it will include that token as a credential when making API requests. To do so, it should transmit the access token to the API as a Bearer credential in an HTTP Authorization header.

Get Access Token

To access the API, an access token must be used.

To request an access token, make a POST call to https://www.systemrequirementslab.com/api/token with the client id and secret provided by System Requirements Lab.

Example access token request:

// Function to fetch an access token.
function getAccessToken()
{
  // Demonstration client ID and secret.
  var credentials = {
    clientId: ": "dXSVY9fRlDZRl69PrOCGd4Oo64rcU5w6",
    clientSecret: "k3VKvrEVDtGPmdwVfVECVkV4ty3GyTNdohFCdgv9q8OX1mtPknh4yD5vIOZlK5j3",
  };

  var tokenUrl = "https://www.systemrequirementslab.com/api/token";

  var options = {
      method: "POST",
      headers: {
          "Content-Type": "application/json",
      },
      body: JSON.stringify(credentials)
  }

  var response = await fetch(tokenUrl, options);

  return await response.json();
}

The access token should be cached for future calls until it expires at which point a new access token may be requested.

Queries

Queries may be performed to get user session, system, and analysis data. Queries are performed using IDs and date ranges.

Date Range
A date range is given to restrict the returned records by date.
User ID
User IDs are supplied by the user or in the start page URL by the caller. They are not generated or used by System Requirements Lab and are provided to callers for user tracking purposes.
Session ID
Session IDs (Tracking ID) identify a single test session. A session comprises session, system, and analysis data.

Date ranges cannot be combined with session, or user IDs.

Use Cases

System Records by User ID

Get all system records for tests performed by a user:

// Get an access token.
var token = await getAccessToken(); // Defined above.

// Fetch the data. Expect just a few records so won't bother with the cursor.
var response = await fetch(
  "https://www.systemrequirementslab.com/api/v2/session/system/1257?user-id=tom-smith",
  {
    method: "GET",
    headers: {
      Authorization: `Bearer ${token.access_token}`,
    },
  }
);
var records = await response.json();

console.log(records);

Analysis Records by Date Range

GET all analyses for tests performed during a date range:

// Get an access token.
var accessToken = await getAccessToken(); // Defined above.

var cursor = ""; // Initial cursor is empty.
var done = false; // Loop exit.

// Many records are possible so will loop over several sets with the cursor.
while (!done) {
  var response = await fetch(
    `https://www.systemrequirementslab.com/api/v2/session/analysis/1257?start=2023-01-01T00:00:00&end=2023-03-27T00:00:00&cursor=${cursor}`,
    {
      method: "GET",
      headers: {
        Authorization: `Bearer ${accessToken}`,
      },
    }
  );

  if (response.status == 200) {
    // Valid records were returned.
    var records = await response.json();
    cursor = records.cursor; 
    console.log(`Got ${records.data.length} records`);
  } else {
    // Response status was '404 not found' (no more records) or other error.
    done = true;
  }
}
